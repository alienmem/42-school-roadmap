📦 COMMON CORE PROJECTS - DETAILED

PROJECT 1: Libft (21 days)

Official Subject PDF
What to Code: 43 Functions Total

Part 1 (23 functions): Recreate standard C library functions

  • ft_isalpha, ft_isdigit, ft_isalnum, ft_isascii, ft_isprint
  • ft_strlen, ft_memset, ft_bzero, ft_memcpy, ft_memmove
  • ft_strlcpy, ft_strlcat, ft_toupper, ft_tolower
  • ft_strchr, ft_strrchr, ft_strncmp, ft_memchr, ft_memcmp
  • ft_strnstr, ft_atoi, ft_calloc, ft_strdup

Part 2 (11 functions): Additional utilities

  • ft_substr, ft_strjoin, ft_strtrim, ft_split
  • ft_itoa, ft_strmapi, ft_striteri
  • ft_putchar_fd, ft_putstr_fd, ft_putendl_fd, ft_putnbr_fd

Bonus (9 functions): Linked list functions

  • ft_lstnew, ft_lstadd_front, ft_lstsize, ft_lstlast
  • ft_lstadd_back, ft_lstdelone, ft_lstclear, ft_lstiter, ft_lstmap
Testing: Search GitHub for "libft tester" or "Tripouille libftTester"

PROJECT 2: ft_printf (14 days)

Official Subject PDF
What to Code: Recreate printf with these conversions:
%c (char), %s (string), %p (pointer), %d/%i (int), %u (unsigned), %x/%X (hex), %% (percent)

PROJECT 3: get_next_line (14 days)

Official Subject PDF
What to Code: Function that reads file line by line
Key concept: Static variables to store state between calls

PROJECT 4: push_swap (14 days)

What to Code: Sort numbers using two stacks with limited operations
Challenge: Sort 100 numbers in ≤700 operations, 500 numbers in ≤5500 operations

PROJECT 5: Philosophers (28 days)

What to Code: Dining philosophers problem with threads and mutexes
Learn: Multithreading, race conditions, deadlock prevention

PROJECT 6: Minishell (14 days, GROUP)

What to Code: Your own shell (like bash)
Features: Pipes, redirections, environment variables, built-ins (echo, cd, pwd, export, unset, env, exit)

🏠 SELF-STUDY STRATEGIES

1. Accountability System

  • Post daily progress in r/42Born2Code
  • Find accountability partner on Discord
  • Track GitHub contribution graph
  • Create public commitment (blog, Twitter)

2. When You're Stuck (48-Hour Rule)

Hours 0-2: Try solving yourself
Hours 2-6: Debug with gdb, add prints, draw on paper
Hours 6-12: Research the concept (not solution!)
Hours 12-24: Take break, sleep on it
Hours 24-48: Start from scratch
After 48h: Ask for help (concepts, not solutions)

3. Debugging Tools

GDB: 
gcc -g program.c -o program
gdb ./program
break main
run
next
print variable
backtrace
Valgrind: 
valgrind --leak-check=full ./program

4. Mental Health

  • Work in sprints: 50min work, 10min break
  • Take days off: Minimum 1 day/week
  • Exercise regularly: Coding is sedentary
  • Sleep 7-8 hours: Brain consolidates during sleep
  • Celebrate small wins: Each working function is progress

5. Progress Tracking

Create a tracking document: 
42 School Progress
==================
Start Date: [DATE]
Projects Completed: [X/20]

Daily Log:
2025-10-15: Completed ft_strlen, stuck on ft_split
2025-10-16: Fixed ft_split! Learned 2D array malloc

Weekly Review (Every Sunday):
- What did I learn?
- What was hardest?
- What's next week's goal?

6. Building Your Portfolio

Since you won't have official certificate:
  • Keep all projects in public GitHub repos
  • Write excellent README files with demos
  • Create portfolio website showcasing projects
  • LinkedIn: "Self-studied École 42 curriculum (1200+ coding hours)"
  • Blog about your journey and learnings

7. Study Techniques

Active Recall: Close docs, write from memory
Spaced Repetition: Review concepts on Day 5, 10, 20, 40
Feynman Technique: Explain concepts simply (as if to 10-year-old)
Learn by Teaching: Answer questions on Reddit/Discord

8. Time Management

Full-Time (40+ hrs/week): 
Monday-Friday:
09:00-13:00  Deep work (hardest problems)
14:00-18:00  Implementation
19:00-21:00  Testing, cleanup

Weekends:
Saturday: Catch-up or get ahead (4-6 hours)
Sunday: REST or light study
Part-Time (20-25 hrs/week): 
Weekday Evenings: 2-3 hours (15 hrs/week)
Weekends: 5-6 hours each day (10 hrs/week)

9. Emergency Motivation

When you want to quit:
  1. "Just 5 minutes" rule: Start for 5min, usually continue
  2. Remember your WHY: Why did you start this?
  3. Look at success stories: Others did it, so can you
  4. Break it down: Just write one function today
  5. Nuclear option: Stop completely for 48h, come back fresh

10. Recommended Resources

  • "The C Programming Language" by K&R
  • "Computer Systems: A Programmer's Perspective"
Videos (for concepts only!):
  • CS50 by Harvard
  • Jacob Sorber (YouTube)
  • CodeVault (YouTube)
